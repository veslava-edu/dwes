    UD2. Prácticas de Sintaxis Básica de PHP

    ¡Hola, equipo de DevCore Solutions!

    Bienvenidos a vuestra primera semana de onboarding. Antes de que os sumerjáis en el código de nuestro proyecto insignia, "Helios", es crucial que todos compartamos una base sólida en PHP. Las siguientes tareas están diseñadas para afinar vuestras habilidades en la sintaxis fundamental del lenguaje. Consideradlas vuestro primer commit de conocimiento. ¡Demostrad que estáis listos para el desafío!


    Ejercicios de Consolidación

    Estos ejercicios son para asegurar que los conceptos básicos están bien asentados. Son el "Hola, Mundo" de un desarrollador profesional.

    El Primer Módulo: Generador de Saludos | Nivel: Fácil

    Objetivo

    Familiarizarse con la declaración de variables, la concatenación de cadenas y el uso de echo para generar una salida HTML dinámica.

    Tarea a realizar

    Crea un script PHP que realice lo siguiente: 1. Declare una variable $nombre_empleado y asígnale tu nombre. 2. Declare una variable $puesto_empleado y asígnale "Desarrollador/a Junior". 3. Usando el operador de concatenación (.), construye un saludo en una tercera variable $mensaje_bienvenida. 4. El saludo debe ser un párrafo HTML (<p>) que diga: "Bienvenido/a a DevCore Solutions, [Tu Nombre]. Tu puesto inicial es: [Puesto]." 5. Imprime la variable $mensaje_bienvenida en el script.

    Aplicación en el Mundo Real

    Esta es la operación más básica en cualquier aplicación web: personalizar la vista para el usuario. Desde un simple saludo hasta la construcción de complejas tarjetas de perfil, todo se basa en estos principios.

    VER Solución. 
    <?php
// 1. Declarar variable de nombre
$nombre_empleado = "Ana García";

// 2. Declarar variable de puesto
$puesto_empleado = "Desarrolladora Junior";

// 3. Construir el mensaje
$mensaje_bienvenida = "<p>Bienvenido/a a DevCore Solutions, " . $nombre_empleado . ". Tu puesto inicial es: " . $puesto_empleado . ".</p>";

// 4. Imprimir el resultado
echo $mensaje_bienvenida;
?>

    Depuración de Código: El Bug en el Contador de Tareas | Nivel: Fácil

    Objetivo

    Desarrollar la habilidad de leer código ajeno e identificar errores de sintaxis comunes, una tarea diaria en el desarrollo en equipo.

    Setup Inicial

    El siguiente script intenta mostrar un resumen de tareas, pero contiene varios errores y no se ejecuta.

    <?php
// Script para contar tareas pendientes del proyecto "Helios"

$nombreProyecto = 'Helios';
$version = 1.0

$num_tareas_pendientes = 5;
$num_tareas_totales = 10;

// Comprobar si no quedan tareas
if ($num_tareas_pendientes = 0) {
    echo "¡Felicidades! No hay tareas pendientes en $nombreProyecto.";
} else {
    echo "Quedan $num_tareas_pendientes tareas de un total de $num_tareas_totales";
}

?>

    Tarea a realizar

    1. Copia el código anterior en un fichero .php.
    2. Identifica y corrige todos los errores para que el script se ejecute correctamente y muestre el mensaje esperado ("Quedan 5 tareas de un total de 10").
    3. Añade comentarios explicando qué errores encontraste y cómo los solucionaste.

    Aplicación en el Mundo Real

    La depuración consume una parte significativa del tiempo de un desarrollador. Identificar rápidamente errores de sintaxis, lógicos (como usar = en lugar de ==) o tipográficos es una habilidad que te hará extremadamente eficiente.

    VER Solución. 
    <?php
// Script para contar tareas pendientes del proyecto "Helios"

$nombreProyecto = 'Helios';
// ERROR 1: Faltaba el punto y coma (;) al final de la línea.
$version = 1.0;

$num_tareas_pendientes = 5;
$num_tareas_totales = 10;

// Comprobar si no quedan tareas
// ERROR 2: Se usaba el operador de asignación (=) en lugar del de comparación (== o ===).
// Esto asignaba 0 a la variable y la condición siempre evaluaba a `false`.
if ($num_tareas_pendientes == 0) {
    echo "¡Felicidades! No hay tareas pendientes en $nombreProyecto.";
} else {
    // ERROR 3: Faltaba el punto y coma (;) al final del `echo`.
    echo "Quedan $num_tareas_pendientes tareas de un total de $num_tareas_totales";
}

?>

    Calculadora de Bonificación Anual | Nivel: Fácil

    Objetivo

    Practicar el uso de operadores aritméticos y la definición de valores inmutables mediante constantes.

    Tarea a realizar

    1. Define una constante llamada FACTOR_BONIFICACION con un valor de 0.10.
    2. Crea una variable $salario_base con un valor de 30000.
    3. Calcula el valor de la bonificación anual ($salario_base * FACTOR_BONIFICACION). Almacena el resultado en una variable $bonificacion.
    4. Calcula el salario total ($salario_base + $bonificacion). Almacena el resultado en $salario_total.
    5. Imprime un resumen claro, por ejemplo:
      • "Salario Base: 30000 €"
      • "Bonificación (10%): 3000 €"
      • "Salario Total: 33000 €"

    Aplicación en el Mundo Real

    Los cálculos financieros son comunes en aplicaciones de negocio, ERPs o ecommerce. Usar constantes para valores que no deben cambiar durante la ejecución (como tipos de IVA, porcentajes de comisiones o factores de conversión) hace el código más legible y seguro.

    VER Solución. 
    <?php
// 1. Definir constante
define("FACTOR_BONIFICACION", 0.10);

// 2. Crear variable de salario
$salario_base = 30000;

// 3. Calcular bonificación
$bonificacion = $salario_base * FACTOR_BONIFICACION;

// 4. Calcular total
$salario_total = $salario_base + $bonificacion;

// 5. Imprimir resumen
echo "Salario Base: " . $salario_base . " €<br>";
echo "Bonificación (" . (FACTOR_BONIFICACION * 100) . "%): " . $bonificacion . " €<br>";
echo "Salario Total: " . $salario_total . " €<br>";
?>

    Informe de Estado del Sistema | Nivel: Fácil

    Objetivo

    Utilizar constantes "mágicas" y funciones nativas de PHP para generar metadatos útiles para la depuración y el logging.

    Tarea a realizar

    1. Crea un script que defina una constante VERSION_SCRIPT con el valor "1.0.2".
    2. El script debe generar un informe que muestre la siguiente información:
      • La ruta completa del fichero que se está ejecutando.
      • La versión del script (usando la constante que creaste).
      • La fecha y hora actual del servidor en el formato Año-Mes-Día Hora:Minutos:Segundos (ej: 2024-10-27 15:30:00).
    3. Investiga el uso de las constantes mágicas de PHP y la función date() para completar la tarea.

    Aplicación en el Mundo Real

    Cuando un error ocurre en producción, los logs son tu única fuente de información. Registrar la ubicación exacta del error (__FILE__, __LINE__), la versión del código y la hora precisa es fundamental para diagnosticar y solucionar problemas rápidamente.

    VER Solución. 
    <?php
// 1. Definir la versión del script
const VERSION_SCRIPT = "1.0.2";

// 2. Generar el informe
echo "--- Informe de Estado de DevCore --<br>";
echo "Ruta del script: " . __FILE__ . "<br>"; // Constante mágica __FILE__
echo "Versión del script: " . VERSION_SCRIPT . "<br>";

// Para la fecha, se usa la función date() con los caracteres de formato correspondientes
// Y: Año con 4 dígitos
// m: Mes con 2 dígitos (01-12)
// d: Día del mes con 2 dígitos (01-31)
// H: Hora en formato 24h (00-23)
// i: Minutos (00-59)
// s: Segundos (00-59)
echo "Fecha y hora del servidor: " . date('Y-m-d H:i:s') . "<br>";
?>

    Tipos de Datos: Inventario de Recursos | Nivel: Fácil

    Objetivo

    Comprender el sistema de tipado dinámico de PHP y cómo verificar el tipo de una variable en tiempo de ejecución.

    Tarea a realizar

    1. Crea un script llamado inventario.php.
    2. Dentro del script, define las siguientes variables que representan los recursos de un proyecto:
      • $nombreProyecto = "Helios"
      • $totalMiembros = 12
      • $presupuesto = 55000.75
      • $despliegueActivo = true
      • $clienteFinal = null
    3. Para cada una de las variables, utiliza la función gettype() para imprimir su tipo de dato junto a su valor. La salida debería ser similar a:
      • Variable nombreProyecto: "Helios" (string)
      • Variable totalMiembros: 12 (integer)

    Aplicación en el Mundo Real

    Aunque PHP es de tipado débil, a menudo necesitas verificar qué tipo de dato estás recibiendo (por ejemplo, de una API o un formulario) para procesarlo correctamente y evitar errores. Funciones como gettype(), is_int(), is_string() son clave en la validación de datos.

    VER Solución. 
    <?php
$nombreProyecto = "Helios";
$totalMiembros = 12;
$presupuesto = 55000.75;
$despliegueActivo = true;
$clienteFinal = null;

echo 'Variable $nombreProyecto: "' . $nombreProyecto . '" (' . gettype($nombreProyecto) . ')<br>';
echo 'Variable $totalMiembros: ' . $totalMiembros . ' (' . gettype($totalMiembros) . ')<br>';
echo 'Variable $presupuesto: ' . $presupuesto . ' (' . gettype($presupuesto) . ')<br>';
// Para los booleanos, un echo simple puede imprimir 1 o nada. var_export es más claro.
echo 'Variable $despliegueActivo: ' . var_export($despliegueActivo, true) . ' (' . gettype($despliegueActivo) . ')<br>';
// Para null, un echo no imprime nada.
echo 'Variable $clienteFinal: ' . var_export($clienteFinal, true) . ' (' . gettype($clienteFinal) . ')<br>';
?>

    Evaluación de Permisos de Usuario | Nivel: Fácil

    Objetivo

    Aplicar el uso de operadores lógicos y de comparación estricta para evaluar condiciones compuestas.

    Tarea a realizar

    1. Imagina que estás controlando el acceso a un panel de administración. Define dos variables:
      • $usuarioLogueado = true;
      • $perfilUsuario = 'admin';
    2. Escribe una expresión lógica que compruebe si ambas condiciones se cumplen: el usuario debe estar logueado Y su perfil debe ser idéntico a 'admin'.
    3. Almacena el resultado true o false de esa expresión en una variable llamada $tieneAcceso.
    4. Imprime el valor de $tieneAcceso. Para que se vea claramente true o false, puedes usar la función var_dump().

    Aplicación en el Mundo Real

    La lógica condicional es el cerebro de cualquier aplicación. Controlar quién puede ver o hacer qué (autorización) es una tarea constante que depende enteramente del uso correcto y seguro de los operadores lógicos y de comparación.

    VER Solución. 
    <?php
$usuarioLogueado = true;
$perfilUsuario = 'admin';

// La expresión combina el valor booleano y una comparación estricta (===)
// usando el operador lógico "Y" (&&)
$tieneAcceso = ($usuarioLogueado === true) && ($perfilUsuario === 'admin');

// var_dump es ideal para depurar porque muestra el tipo y el valor
echo "El usuario tiene acceso al panel de administración: ";
var_dump($tieneAcceso);
?>


    Ejercicios de Refuerzo

    Estos ejercicios te guiarán a través de problemas un poco más complejos. Se proporcionan pistas para orientarte.

    Refactorizando el Módulo de Configuración | Nivel: Medio

    Objetivo

    Aprender a refactorizar código, reemplazando variables mutables por constantes y aplicando comparaciones estrictas para aumentar la robustez y seguridad del script.

    Setup Inicial

    Este script de configuración funciona, pero no sigue las mejores prácticas. Usa variables para valores que deberían ser fijos y comparaciones débiles.

    <?php
// Configuración del entorno de la aplicación "Helios"

$app_env = 'development'; // Podría ser 'production'
$db_host = 'localhost';
$db_user = 'dev_user';
$debug_mode = '1'; // El '1' se trata como true

if ($app_env == 'production') {
    echo "MODO PRODUCCIÓN: Conectando a la base de datos en $db_host...<br>";
}

if ($debug_mode == true) {
    echo "MODO DEBUG ACTIVADO.<br>";
}
?>

    Tarea a realizar

    1. Reemplaza las variables $db_host y $db_user por constantes (DB_HOST, DB_USER), ya que estos valores no deberían cambiar nunca en tiempo de ejecución.
    2. Cambia la variable $debug_mode para que use un valor booleano (true) en lugar de un string ('1').
    3. Modifica las condiciones if para que utilicen el operador de comparación idéntico (===).
    4. Ajusta el resto del código para que siga funcionando con los cambios realizados.


    Pista 1 Recuerda la convención para nombrar constantes en mayúsculas. Puedes usar const o define().
    Pista 2 Al usar ===, no solo comparas el valor, sino también el tipo. 1 == true es verdadero, pero 1 === true es falso. Asegúrate de que los valores que comparas sean del mismo tipo.

    Aplicación en el Mundo Real

    Refactorizar es el proceso de mejorar el código existente sin cambiar su comportamiento externo. Es una tarea clave para mantener la calidad, legibilidad y seguridad del software a lo largo del tiempo, especialmente en proyectos grandes como "Helios".

    VER Solución. 
    <?php
// Configuración del entorno de la aplicación "Helios"

// 1. Valores de BBDD como constantes
const DB_HOST = 'localhost';
const DB_USER = 'dev_user';

$app_env = 'development'; // Podría ser 'production'

// 2. Debug mode como booleano
$debug_mode = true;

// 3. Condición con comparación ESTRICTA
if ($app_env === 'production') {
    // 4. Usar la constante en el string
    echo "MODO PRODUCCIÓN: Conectando a la base de datos en " . DB_HOST . "...<br>";
}

// 3. Condición con comparación ESTRICTA
if ($debug_mode === true) {
    echo "MODO DEBUG ACTIVADO.<br>";
}
?>

    Gestor Dinámico de Perfiles | Nivel: Medio

    Objetivo

    Utilizar el operador de fusión de null (??) como una forma moderna y concisa de asignar valores por defecto.

    Setup Inicial

    Tu script debe obtener el nombre de usuario de una petición GET. A veces, este valor no existirá. Para simularlo, puedes crear la variable $_GET manualmente.

    <?php
// Simulación 1: El usuario viene en la URL (ej: /perfil.php?usuario=Carlos)
$_GET['usuario'] = 'Carlos';

// Simulación 2: El usuario no se especifica (ej: /perfil.php)
// unset($_GET['usuario']); // Descomenta esta línea para probar el caso nulo
?>

    Tarea a realizar

    1. Usando el setup anterior, crea una variable $nombreUsuario.
    2. Asigna a $nombreUsuario el valor de $_GET['usuario'] si existe y no es null.
    3. Si $_GET['usuario'] no existe o es null, asigna a $nombreUsuario el valor por defecto 'Invitado'.
    4. No uses una estructura if/else. Debes resolverlo en una sola línea de asignación.
    5. Imprime el saludo final: echo "Bienvenido al dashboard de Helios, $nombreUsuario.";
    6. Prueba tu script con y sin la variable $_GET['usuario'] definida.


    Pista El material didáctico menciona un operador introducido en PHP 7 que es perfecto para esto. Su sintaxis es valor_a_probar ?? valor_por_defecto.

    Aplicación en el Mundo Real

    Este operador es extremadamente útil y común. Se usa constantemente para manejar datos de formularios, parámetros de URLs ($_GET), o datos de APIs, donde muchos campos pueden ser opcionales. Simplifica enormemente el código y lo hace más legible.

    VER Solución. 
    <?php
// Simulación 1: El usuario viene en la URL
$_GET['usuario'] = 'Carlos';

// La solución en una sola línea usando el operador de fusión de null
$nombreUsuario = $_GET['usuario'] ?? 'Invitado';

echo "Bienvenido al dashboard de Helios, $nombreUsuario.<br>"; // Imprime "Carlos"

// ----------------------------------------------------

// Simulación 2: El usuario no se especifica
unset($_GET['usuario']); // Eliminamos la variable para simular que no existe

$nombreUsuario = $_GET['usuario'] ?? 'Invitado';

echo "Bienvenido al dashboard de Helios, $nombreUsuario.<br>"; // Imprime "Invitado"
?>

    El Modificador de Métricas: ¿Valor o Referencia? | Nivel: Medio

    Objetivo

    Comprender la diferencia práctica entre el paso por valor y el paso por referencia, y saber cuándo utilizar este último para modificar variables externas desde una función.

    Tarea a realizar

    1. Crea una variable $porcentaje_progreso y asígnale el valor 25.
    2. Define una función avanzar_proyecto($progreso_actual). Dentro de esta función, incrementa el valor del parámetro recibido en 10.
    3. Llama a la función avanzar_proyecto() pasándole $porcentaje_progreso.
    4. Imprime el valor de $porcentaje_progreso después de llamar a la función.
    5. Observa el resultado. Te darás cuenta de que la variable original no ha cambiado.
    6. Modifica la definición de la función avanzar_proyecto para que acepte el parámetro por referencia, de modo que la variable original sí sea modificada.
    7. Vuelve a ejecutar el script y comprueba que ahora el valor final impreso es 35.


    Pista Para que una función acepte un parámetro por referencia, necesitas añadir un único carácter especial justo antes del nombre del parámetro en la declaración de la función.

    Aplicación en el Mundo Real

    Aunque se debe usar con cuidado, el paso por referencia es útil para modificar directamente grandes estructuras de datos (como arrays u objetos) sin la sobrecarga de memoria que supondría copiarlos y devolverlos. Entender este mecanismo es clave para optimizar el rendimiento en ciertos escenarios.

    VER Solución. 
    <?php
// PASO 1: Implementación inicial (paso por valor)
echo "--- DEMO PASO POR VALOR ---<br>";
$porcentaje_progreso = 25;

function avanzar_proyecto_valor($progreso_actual) {
    $progreso_actual += 10;
    echo "Progreso dentro de la función: $progreso_actual<br>";
}

avanzar_proyecto_valor($porcentaje_progreso);
echo "Progreso fuera de la función: $porcentaje_progreso<br>"; // Sigue siendo 25

echo "<hr>";

// PASO 2: Solución correcta (paso por referencia)
echo "--- DEMO PASO POR REFERENCIA ---<br>";
$porcentaje_progreso_ref = 25;

// Se añade el ampersand (&) para indicar el paso por referencia
function avanzar_proyecto_referencia(&$progreso_actual) {
    $progreso_actual += 10;
    echo "Progreso dentro de la función: $progreso_actual<br>";
}

avanzar_proyecto_referencia($porcentaje_progreso_ref);
echo "Progreso fuera de la función: $porcentaje_progreso_ref<br>"; // Ahora es 35

?>


    Ejercicios de Ampliación

    Estos son retos abiertos que requieren investigación y pensamiento crítico, preparándote para los problemas complejos del mundo real.

    Investigación: El Operador 'Nave Espacial' y la Ordenación de Proyectos | Nivel: Alto

    Objetivo

    Investigar funciones de array avanzadas (usort) y aplicar operadores modernos (<=>) para resolver un problema de ordenación complejo de forma elegante y eficiente.

    Tarea a realizar

    En el panel "Helios", necesitamos ordenar una lista de proyectos según su prioridad. Un array de proyectos podría tener esta estructura: 

    $proyectos = [
    ['nombre' => 'Helios Core', 'prioridad' => 2],
    ['nombre' => 'API Gateway', 'prioridad' => 1],
    ['nombre' => 'Frontend Refactor', 'prioridad' => 3],
    ['nombre' => 'Docs Migration', 'prioridad' => 2],
];
    1. Investiga la función de PHP usort(). ¿Para qué sirve? ¿Qué tipo de argumento espera como segundo parámetro? 2. Escribe una función de comparación que usort() pueda usar. Esta función recibirá dos proyectos ($a y $b) y debe devolver -1, 0 o 1. 3. Utiliza el operador nave espacial (<=>) dentro de tu función de comparación para determinar el orden basado en la clave 'prioridad'. La ordenación debe ser ascendente (menor prioridad primero). 4. Aplica usort() a tu array de $proyectos con tu función de comparación. 5. Imprime el array resultante (puedes usar print_r o var_dump) para verificar que los proyectos están ordenados correctamente por prioridad.

    Revisión y Discusión

    Prepara una breve explicación (2 minutos) para tus compañeros de equipo. Describe cómo usort delega la lógica de comparación a tu función, y por qué el operador <=> es la herramienta perfecta para esta tarea, ahorrando varias líneas de código if/else.

    Aplicación en el Mundo Real

    Casi todas las aplicaciones complejas necesitan ordenar datos de formas personalizadas: por fecha, por precio, por relevancia, por apellidos... Dominar usort y los operadores de comparación te da el poder de implementar cualquier lógica de ordenación que el negocio requiera.

    VER Solución. 
    <?php
$proyectos = [
    ['nombre' => 'Helios Core', 'prioridad' => 2],
    ['nombre' => 'API Gateway', 'prioridad' => 1],
    ['nombre' => 'Frontend Refactor', 'prioridad' => 3],
    ['nombre' => 'Docs Migration', 'prioridad' => 2],
];

echo "Array original:<br>";
print_r($proyectos);

// usort() permite al usuario definir su propia lógica de comparación.
// Espera un "callable" (como el nombre de una función o una función anónima)
// como segundo argumento.

// La función de comparación debe aceptar dos argumentos ($a, $b)
// y devolver:
// < 0 si $a es menor que $b
// = 0 si $a es igual a $b
// > 0 si $a es mayor que $b
// El operador nave espacial ($a <=> $b) hace exactamente esto.

usort($proyectos, function($a, $b) {
    // Comparamos los proyectos basándonos en su clave 'prioridad'
    return $a['prioridad'] <=> $b['prioridad'];
});

echo "<hr>Array ordenado por prioridad:<br>";
print_r($proyectos);

    Revisión de Seguridad: El Peligro del Tipado Débil | Nivel: Alto

    Objetivo

    Analizar código vulnerable, investigar una categoría específica de vulnerabilidad de seguridad (Type Juggling) y aplicar las mejores prácticas para solucionarla.

    Setup Inicial

    Un desarrollador anterior dejó este fragmento de código en un sistema antiguo para una "autenticación rápida". La contraseña correcta es supersecret.

    <?php
// Contraseña "segura" almacenada en el código (¡mala práctica!)
$password_real = 'supersecret';

// Simulación de un dato enviado por un usuario
// Prueba a cambiar el valor de $_POST['password']
$_POST['password'] = '0'; 

// Comprobación de autenticación vulnerable

    echo "Acceso concedido.";

    echo "Acceso denegado.";
}
?>

    Tarea a realizar

    2. Demuestra la vulnerabilidad: Modifica el valor de $_POST['password'] en el script. Encuentra un valor que, sin ser la contraseña correcta, haga que la condición if se evalúe como true. Explica por qué ocurre esto.
    3. Refactoriza el código: Corrige la vulnerabilidad para que la comparación sea segura y estricta, previniendo el "type juggling".

    Aplicación en el Mundo Real

    Las vulnerabilidades de Type Juggling han sido la causa de fallos de seguridad críticos en aplicaciones reales, incluyendo CMS populares en el pasado. Forzar siempre la comparación estricta (===) es una de las reglas de oro de la programación segura en PHP.

    VER Solución.

    1. Investigación

    Una vulnerabilidad de "Type Juggling" ocurre cuando se usa el operador de comparación no estricto (==) en PHP. Este operador intenta convertir los tipos de datos de los operandos para que coincidan antes de compararlos. Un atacante puede aprovechar este comportamiento para sortear comprobaciones de seguridad. Por ejemplo, al comparar un string con un número, PHP intentará convertir el string a un número. El string 'supersecret' se convierte al número 0, porque no empieza con dígitos.

    2. Demostración de la vulnerabilidad

    Si un atacante envía el valor 0 (el número entero) o '0' (el string), la comparación se convierte en 0 == 'supersecret'. PHP convierte 'supersecret' a 0, resultando en 0 == 0, que es true.

    <?php
$password_real = 'supersecret';

// El atacante envía el número 0.
$_POST['password'] = 0; 

// La comparación: 0 == 'supersecret' -> 0 == 0 -> true
if ($_POST['password'] == $password_real) {
    echo "Acceso concedido. (¡VULNERABILIDAD EXPLOTADA!)";
} else {
    echo "Acceso denegado.";
}
?>

    3. Código Refactorizado

    La solución es usar siempre el operador de comparación idéntico (===), que comprueba tanto el valor como el tipo, sin realizar conversiones.

    <?php
$password_real = 'supersecret';

$_POST['password'] = 0; 

// Con ===, la comparación es: (int) 0 === (string) 'supersecret'
// Los tipos no coinciden, por lo que el resultado es inmediatamente `false`.
if ($_POST['password'] === $password_real) {
    echo "Acceso concedido.";
} else {
    echo "Acceso denegado. (CÓDIGO SEGURO)";
}
?>

    Aplicación en el Mundo Real

    En el procesamiento de grandes volúmenes de datos (Big Data, ETLs, procesamiento de logs), la gestión de la memoria es crítica. Saber cuándo evitar copias innecesarias de grandes arrays puede ser la diferencia entre una aplicación que funciona y una que agota la memoria del servidor y se cae.

    VER Solución. 
    <?php
// Generamos un array grande
$logs_valor = [];
for ($i = 0; $i < 100000; $i++) {
    $logs_valor[] = "  Evento de Sistema " . ($i + 1) . "  ";
}
$logs_referencia = $logs_valor; // Creamos una copia para el segundo ejemplo

// --- 1. Enfoque por VALOR (devolviendo una copia) ---
function limpiarLogsPorValor(array $listaLogs): array
{
    // Dentro de esta función, $listaLogs es una COPIA del original.
    foreach ($listaLogs as $indice => $valor) {
        $listaLogs[$indice] = strtolower(trim($valor));
    }
    // Devolvemos la copia modificada.
    return $listaLogs;
}

// En este punto, existen DOS copias del array en memoria:
// $logs_valor (original) y la copia devuelta por la función.
$logs_valor = limpiarLogsPorValor($logs_valor);


// --- 2. Enfoque por REFERENCIA (modificando el original) ---
function limpiarLogsPorReferencia(array &$listaLogs): void
{
    // Aquí, $listaLogs es una REFERENCIA (un alias) al array original.
    // No se crea ninguna copia grande.
    foreach ($listaLogs as $indice => $valor) {
        $listaLogs[$indice] = strtolower(trim($valor));
    }
    // No se necesita devolver nada, la variable original ya ha sido modificada.
}

// En este punto, solo existe UN array en memoria.
// La función opera directamente sobre él.
limpiarLogsPorReferencia($logs_referencia);


/*
--- 3. ANÁLISIS ---

El enfoque de **Paso por Valor** es más seguro y predecible. La función no tiene
"efectos secundarios", es decir, no modifica variables fuera de su propio
ámbito. Sin embargo, para un array de 100,000 elementos, implica duplicar la
memoria utilizada temporalmente: una para el array original y otra para la
copia que se crea y se devuelve.

El enfoque de **Paso por Referencia** es mucho más eficiente en memoria en este
caso. No se crea una copia del array, la función opera sobre los datos
originales. Esto es ideal para grandes conjuntos de datos donde la memoria
es una preocupación. La desventaja es que el código puede ser más difícil de
razonar, ya que una función está modificando un estado externo de forma
implícita.

**Conclusión para este caso:** Dado el gran tamaño del array, el paso por
referencia es una optimización justificada y preferible, siempre que
el nombre de la función (ej. 'modificarLogs' o 'limpiarLogsInSitu') deje
claro su comportamiento.

Nota: Las versiones modernas de PHP usan un mecanismo "Copy on Write", por lo que
la copia no se realiza hasta que el array es modificado dentro de la función.
Aun así, al modificarlo, la copia se realiza, por lo que la sobrecarga de
memoria en el enfoque por valor sigue siendo un factor.
*/
