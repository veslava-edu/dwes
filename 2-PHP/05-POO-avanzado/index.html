    Conceptos Adicionales y Casos de Uso

    ¡Excelente trabajo, equipo! En el Tema 5: Programación Orientada a Objetos en PHP, construimos los cimientos de nuestra fábrica de software. Ya dominamos los planos (clases), los productos (objetos), las líneas de ensamblaje (herencia) y los contratos de calidad (interfaces y clases abstractas).

    Ahora, vamos a profundizar en la maquinaria de precisión. Este tema cubre conceptos más específicos pero igualmente cruciales que nos darán un control total sobre el ciclo de vida, la identidad y el comportamiento de nuestros objetos. Son las herramientas que distinguen a un operario de un ingeniero jefe de la fábrica.

    Clonación de Objetos: Creando Copias Reales

    En PHP, cuando asignas un objeto a una nueva variable, no estás creando una copia. Estás creando otra "etiqueta" o referencia que apunta al mismo objeto en memoria. Esto puede llevar a efectos secundarios inesperados. La clonación nos permite crear una copia genuinamente nueva e independiente de un objeto existente.

    Definición: Clonación

    La clonación es el proceso de crear un nuevo objeto que es una copia de un objeto existente. En PHP, esto se logra con la palabra clave clone. Por defecto, clone realiza una copia superficial (shallow copy): las propiedades del objeto son copiadas. Sin embargo, si alguna propiedad es a su vez un objeto, esa propiedad seguirá siendo una referencia al objeto original.

    Para obtener un control total sobre el proceso de clonación (por ejemplo, para realizar una copia profunda donde los objetos internos también se clonen), PHP nos proporciona el método mágico __clone(). Este método se llama automáticamente en el objeto recién clonado después de que la copia inicial se ha realizado.

    <?php
class FichaPersonaje
{
    public string $nombre;
    public Atributos $atributos;

    public function __construct(string $nombre, int $fuerza, int $destreza)
    {
        $this->nombre = $nombre;
        $this->atributos = new Atributos($fuerza, $destreza);
    }

    // Este método se invoca en el objeto clonado
    public function __clone()
    {
        // Forzamos que la propiedad 'atributos' también se clone.
        // Esto se conoce como una copia profunda (deep copy).
        $this->atributos = clone $this->atributos;
        echo "<p><strong>¡Ficha de {$this->nombre} clonada profundamente!</strong></p>";
    }
}

class Atributos
{
    public int $fuerza;
    public int $destreza;

    public function __construct(int $fuerza, int $destreza)
    {
        $this->fuerza = $fuerza;
        $this->destreza = $destreza;
    }
}

$guerrero = new FichaPersonaje("Krom", 18, 12);

// $ladron = $guerrero; // ¡ERROR! Esto es solo una referencia.
$ladron = clone $guerrero; // Creamos una copia real.
$ladron->nombre = "Lyss";

// Modificamos los atributos del ladrón
$ladron->atributos->destreza = 20;

// Gracias a __clone(), los atributos del guerrero permanecen intactos.
echo "Atributos de {$guerrero->nombre}: FUE {$guerrero->atributos->fuerza}, DES {$guerrero->atributos->destreza}"; // Salida: FUE 18, DES 12
echo "<br>";
echo "Atributos de {$ladron->nombre}: FUE {$ladron->atributos->fuerza}, DES {$ladron->atributos->destreza}"; // Salida: FUE 18, DES 20

    A reflexionar

    Si comentaras el método __clone() en la clase FichaPersonaje, ¿cuál sería la salida del script? ¿Por qué?

    Comparación de Objetos: Identidad vs. Igualdad

    Una pregunta fundamental al trabajar con objetos es: ¿cuándo son dos variables de objeto "iguales"? PHP nos ofrece dos operadores para responder a esta pregunta, y es vital entender su diferencia.

    • Operador de Igualdad (==): Dos objetos son considerados iguales si son instancias de la misma clase y todas sus propiedades tienen los mismos valores.
    • Operador de Identidad (===): Dos objetos son idénticos si, y solo si, ambas variables apuntan exactamente a la misma instancia en memoria.
    Comparación == (Igualdad) === (Identidad)
    ¿Qué compara? Clase y valores de propiedades Referencia en memoria
    Significado "¿Son objetos del mismo tipo con el mismo estado?" "¿Son literalmente el mismo objeto?" 
    <?php
class Color
{
    public function __construct(public int $r, public int $g, public int $b) {}
}

$rojo1 = new Color(255, 0, 0);
$rojo2 = new Color(255, 0, 0);
$rojo3 = $rojo1; // $rojo3 es una referencia a $rojo1
$azul  = new Color(0, 0, 255);

var_dump($rojo1 == $rojo2); // bool(true) -> Misma clase, mismas propiedades y valores.
var_dump($rojo1 === $rojo2); // bool(false) -> Son dos objetos distintos en memoria.

var_dump($rojo1 == $rojo3); // bool(true) -> Son iguales.
var_dump($rojo1 === $rojo3); // bool(true) -> Apuntan al mismo objeto.

var_dump($rojo1 == $azul); // bool(false) -> Tienen valores de propiedad diferentes.

    La regla de oro

    Usa === cuando necesites saber si dos variables manipulan el mismo objeto. Usa == cuando te importe si dos objetos, aunque sean instancias diferentes, representan el mismo "valor" o estado.

    Miembros Estáticos: Propiedades y Métodos de Clase

    Hasta ahora, todas las propiedades y métodos que hemos visto pertenecen a una instancia específica de una clase (un objeto). Para acceder a ellos, siempre necesitamos un objeto ($miObjeto->miMetodo()). Sin embargo, a veces necesitamos propiedades o métodos que pertenezcan a la clase en sí, no a una instancia individual. Estos son los miembros estáticos.

    Definición: Miembro Estático

    Un miembro estático (propiedad o método), declarado con la palabra clave static, pertenece a la clase en lugar de a una instancia. Se puede acceder a él directamente desde la clase usando el Operador de Resolución de Ámbito (::), también conocido como Paamayim Nekudotayim. Dentro de un método estático, no se puede usar la pseudo-variable $this porque no hay una instancia de objeto asociada.

    Los miembros estáticos son útiles para: * Contadores (ej: cuántas instancias de una clase se han creado). * Métodos de utilidad o "helpers" que no dependen del estado de un objeto. * Implementar patrones de diseño como Factory o Singleton.

    <?php
class GestorConexiones
{
    // Propiedad estática para contar las conexiones activas
    public static int $conexionesActivas = 0;

    // Propiedad estática para almacenar una instancia (Patrón Singleton)
    private static ?self $instancia = null;

    // El constructor es privado para evitar que se creen instancias con `new`
    private function __construct()
    {
        self::$conexionesActivas++;
        echo "<p>Nueva conexión establecida. Activas: " . self::$conexionesActivas . "</p>";
    }

    // Método estático "Factory" para obtener la única instancia
    public static function obtenerInstancia(): self
    {
        if (self::$instancia === null) {
            self::$instancia = new self();
        }
        return self::$instancia;
    }

    // Un método de instancia normal
    public function consultar(string $query): string
    {
        return "Ejecutando la consulta: '$query'";
    }

    // El destructor también debe actualizar el contador estático
    public function __destruct()
    {
        self::$conexionesActivas--;
        echo "<p>Conexión cerrada. Activas: " . self::$conexionesActivas . "</p>";
    }
}

// No podemos hacer esto:
// $conexion = new GestorConexiones(); // Error Fatal: constructor privado

// Obtenemos la instancia a través del método estático
$conexion1 = GestorConexiones::obtenerInstancia();
echo $conexion1->consultar("SELECT * FROM usuarios");
echo "<br>";

// Si lo pedimos de nuevo, nos devuelve la MISMA instancia
$conexion2 = GestorConexiones::obtenerInstancia();
var_dump($conexion1 === $conexion2); // bool(true)

// Podemos acceder a la propiedad estática directamente desde la clase
echo "<p>Total de conexiones activas: " . GestorConexiones::$conexionesActivas . "</p>"; // Salida: 1

    Las palabras clave self, parent y static

    Dentro de una clase, para hacer referencia a la propia clase, tienes varias opciones: * self::: Se refiere a la clase actual, donde se escribió el código. * parent::: Se refiere a la clase padre en un contexto de herencia. * static::: Se refiere a la clase que fue llamada en tiempo de ejecución. Esto es para el Enlace Estático en Tiempo de Ejecución (Late Static Binding), un concepto más avanzado pero muy potente.

    Iteración de Objetos: Recorriendo sus Propiedades

    Por defecto, PHP permite iterar sobre un objeto usando un bucle foreach como si fuera un array asociativo. Este comportamiento nos da una forma sencilla de inspeccionar las propiedades de un objeto.

    Cuando se itera sobre un objeto, el bucle foreach recorrerá todas las propiedades que sean visibles en el ámbito actual.

    • Desde fuera de la clase, solo se iterará sobre las propiedades public.
    • Desde dentro de un método de la clase, se iterará sobre las propiedades public, protected y private.
    <?php
class Configuracion
{
    public string $nombreApp = "Mi Fábrica de Código";
    public string $version = "1.0.2";
    protected string $entorno = "producción";
    private string $apiKey = "secreto-12345-xyz";

    public function mostrarVisibles()
    {
        echo "<strong>Propiedades visibles desde DENTRO de la clase:</strong><br>";
        foreach ($this as $clave => $valor) {
            echo "- $clave: $valor <br>";
        }
    }
}

$config = new Configuracion();

echo "<strong>Propiedades visibles desde FUERA de la clase:</strong><br>";
foreach ($config as $clave => $valor) {
    echo "- $clave: $valor <br>";
}

echo "<hr>";

$config->mostrarVisibles();

    Personalizando la Iteración

    Si necesitas un control más fino sobre cómo se itera tu objeto (por ejemplo, para calcular valores al vuelo o iterar sobre datos privados de forma controlada), puedes hacer que tu clase implemente la interfaz Iterator o IteratorAggregate de PHP. Esto te permite definir exactamente qué sucede en cada paso del bucle foreach.
